# Command Injection

เคสนี้คือโปรแกรมเอาข้อมูลจากผู้ใช้ไปต่อเป็น command แล้วสั่งรันต่ออีกที

ถ้าพูดแบบภาษาคน:

- ผู้ใช้กรอกอะไรบางอย่างเข้ามา
- โค้ดเอาค่านั้นไปประกอบ string
- แล้วเรียก `system()` หรือ `popen()`
- ถ้าไม่มีการคุม input ดีพอ ก็มีโอกาสโดนแทรกคำสั่ง

## สัญญาณที่มักเจอ

- มี `fgets`, `gets`, `scanf`, argv หรือ input อื่น ๆ
- มี `snprintf` หรือ `sprintf` เอาไว้ประกอบคำสั่ง
- ปลายทางไปจบที่ `system`, `popen`, `exec*`
- ใน string มี pattern ประมาณ `echo %s`, `cat %s`, `sh -c ...`

## เวลาระบบเจอ path แบบนี้ มันน่ากังวลเพราะ

- input จาก user ยังมีอิทธิพลต่อ command
- shell metacharacter อย่าง `;`, `&&`, `|`, `` ` ``, `$()` อาจทำให้ command เปลี่ยนความหมายได้
- ถึงมี guard บางอย่างก็ไม่ได้แปลว่าปลอดภัยเสมอ

## แนวทางแก้

- ถ้าเลี่ยง shell ได้ ให้เลี่ยงเลย
- ถ้าต้องใช้จริง ให้ทำ allow-list
- อย่าเอา input ดิบ ๆ ไปต่อ string แล้วรันทันที
- แยก data ออกจาก command ให้ชัด
