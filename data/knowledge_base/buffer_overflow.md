# Buffer Overflow

อันนี้คือเคสคลาสสิกมากของงานสาย pwn หรือ secure coding

ภาพง่าย ๆ คือ:

- มีข้อมูลจากข้างนอกเข้ามา
- โปรแกรมเช็กขนาดไม่ดีพอ
- แล้วเอาไป copy ลง buffer
- สุดท้ายข้อมูลล้น

## สัญญาณที่มักเจอ

- ใช้ `strcpy`, `strcat`, `memcpy` แบบไม่เช็กขนาด
- input ยาวแค่ไหนก็รับหมด
- buffer ปลายทางเล็ก แต่ข้อมูลต้นทางอาจยาวกว่า
- ไม่มี `if` หรือ size check ก่อนถึง sink

## ถ้าระบบเจอ path แบบนี้ มักจะตีความว่า

- จุดเริ่มต้นน่าจะเป็น input boundary
- จุดอันตรายคือ copy operation
- ความเสี่ยงจะยิ่งสูงถ้าเป็น stack buffer และไม่มี guard

## แนวทางแก้แบบง่ายและตรง

- เปลี่ยนไปใช้ API ที่มีการกำหนดขนาดชัดเจน
- เช็กความยาวก่อน copy
- ลดการใช้ raw char buffer ถ้าไม่จำเป็น
- อย่าคิดว่า input จะมาสั้นเสมอ
