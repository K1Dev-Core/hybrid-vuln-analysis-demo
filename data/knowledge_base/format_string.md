# Format String

เคสนี้ชอบหลุดในโค้ด C ตอนคนเขียนเผลอเอา input ของผู้ใช้ไปเป็น format string ตรง ๆ

ตัวอย่างอารมณ์ประมาณนี้:

```c
printf(user_input);
```

ปัญหาคือ `printf` จะไม่ได้มองว่ามันเป็นข้อความธรรมดา แต่มันจะพยายามตี `%x`, `%s`, `%n` เป็น format specifier ด้วย

## สัญญาณที่มักเจอ

- `printf(user_input)` หรือทรงใกล้เคียงกัน
- ไม่มี format string คงที่ เช่น `"%s"`
- input จาก user ไหลตรงเข้าฟังก์ชันตระกูล `printf`

## ทำไมมันอันตราย

- อาจทำให้ข้อมูลในหน่วยความจำรั่ว
- บางกรณีรุนแรงถึงขั้นเขียนหน่วยความจำได้
- มักดูเหมือนโค้ดสั้น ๆ ธรรมดา แต่จริง ๆ เสี่ยงมาก

## แนวทางแก้

- ใช้ format string คงที่เสมอ
- ถ้าจะพิมพ์ข้อความจากผู้ใช้ ให้ส่งเป็น data argument
- ตัวอย่างที่ปลอดภัยกว่า:

```c
printf("%s", user_input);
```
